# Migration Guide: Upgrading to Google Gemini APIs & Multimodal Live Voice

This document details the architectural blueprint, pricing analysis, and code-level transition plan for replacing Anthropic Claude and the traditional voice pipeline with the Google Gemini API suite, specifically leveraging the **Gemini Multimodal Live API** for ultra-low latency, voice-to-voice mock interviews.

---

## 1. Executive Summary

The current mock-interview implementation relies on a sequential text-based pipeline:
$$\text{Candidate Voice} \xrightarrow{\text{STT (Browser)}} \text{Transcript Text} \xrightarrow{\text{API}} \text{Claude (Text)} \xrightarrow{\text{API}} \text{ElevenLabs (TTS)} \xrightarrow{\text{Audio Playback}}$$
This approach introduces significant latency (**3.0s to 5.5s** per turn) and incurs substantial ElevenLabs synthesis costs.

By migrating to the **Google Gemini Multimodal Live API**, we transition to a native audio-to-audio streaming architecture:
$$\text{Candidate Audio} \xrightarrow{\text{WebSocket}} \text{Gemini Live (Audio-in/Audio-out)} \xrightarrow{\text{WebSocket}} \text{Audio Stream Playback}$$
This reduces turn-taking latency to sub-second levels (**0.8s to 1.5s**), implements native voice activity detection (VAD), handles user interruptions ("barge-in") out of the box, and drastically reduces recurring operational costs.

## 4. Live Voice-to-Voice Architecture

The Gemini Multimodal Live API requires a stateful WebSocket connection. We can implement this in one of two configurations:

### Option A: Direct Client-to-API WebSocket (Recommended for Prototyping/Latency)
The React client connects directly to Gemini using a secure, short-lived ephemeral token generated by our Next.js API.

```
Candidate Browser <================= WSS (Audio In/Out) =================> Google Gemini Live
        │                                                                           ▲
        │ (Save Transcript / Update State)                                          │ (Tools/Function Calls)
        ▼                                                                           │
Next.js REST API (/api/interviews/[id]/turn) =======================================┘
```

* **Pros**: Lowest possible latency (no intermediate proxy hops).
* **Cons**: Relies on Client-side SDK logic. Credit accounting must be triggered via Gemini's Function Calling (the model calls our tool after completing a question/answer pair).

### Option B: Server-side WebSocket Proxy (Recommended for Security/State Control)
The React client connects to our dedicated Fastify/Node.js WebSocket proxy, which manages the pipeline.

```
Candidate Browser <==== WSS ====> Fastify Proxy Server <==== WSS ====> Google Gemini Live
                                         │
                                         ├─► Ledger & Credit Deduction (packages/db)
                                         └─► State Tracking (packages/orchestrator)
```

* **Pros**: Centralized control. Transparent logging of every audio packet. Direct credit billing (`applyCreditDelta`) on every turn.
* **Cons**: Introduces a minor network proxy hop (typically <50ms overhead if hosted in the same region). Requires deploying the Fastify server.

## 5. Step-by-Step Technical Migration

### Step 5.1: Replace Anthropic SDK with Google Gen AI SDK
Uninstall `@anthropic-ai/sdk` and install the official Google Gen AI SDK:
```bash
pnpm remove @anthropic-ai/sdk
pnpm add @google/genai
```

### Step 5.2: Refactor AI Client (`packages/ai-core/src/client.ts`)
Rewrite `client.ts` to instantiate `GoogleGenAI` and implement `extractStructured` using Gemini's native JSON output controls.

```typescript
import { GoogleGenAI, Type } from "@google/genai";
import type { z } from "zod";
import { zodToJsonSchema } from "zod-to-json-schema";

let aiInstance: GoogleGenAI | null = null;

function getGeminiClient(): GoogleGenAI {
  if (!aiInstance) {
    if (!process.env.GEMINI_API_KEY) {
      throw new Error("GEMINI_API_KEY is not set");
    }
    aiInstance = new GoogleGenAI({ apiKey: process.env.GEMINI_API_KEY });
  }
  return aiInstance;
}

export const FLASH_MODEL = "gemini-2.5-flash";
export const PRO_MODEL = "gemini-2.5-pro";
export const LIVE_MODEL = "gemini-2.0-flash-exp"; // Or latest live audio model

export async function extractStructured<T>(params: {
  system: string;
  prompt: string;
  zodSchema: z.ZodType<T>;
  model?: string;
}): Promise<T> {
  const client = getGeminiClient();
  const jsonSchema = zodToJsonSchema(params.zodSchema);

  const response = await client.models.generateContent({
    model: params.model ?? FLASH_MODEL,
    contents: params.prompt,
    config: {
      systemInstruction: params.system,
      responseMimeType: "application/json",
      responseSchema: jsonSchema as any, // Cast to Gemini Schema layout
    },
  });

  const text = response.text;
  if (!text) {
    throw new Error("Gemini returned an empty response");
  }

  return params.zodSchema.parse(JSON.parse(text));
}
```

### Step 5.4: Live Interview WebSocket Implementation
To stream audio back and forth, build a WebSocket connection handler. When initiating the Live API WebSocket, send the initial `setup` message containing system instructions, tools, and the target audio output format:

```json
{
  "setup": {
    "model": "models/gemini-2.0-flash-exp",
    "generationConfig": {
      "responseModalities": ["AUDIO"],
      "speechConfig": {
        "voiceConfig": {
          "prebuiltVoiceConfig": {
            "voiceName": "Puck" 
          }
        }
      }
    },
    "systemInstruction": {
      "parts": [{ "text": "You are a senior technical interviewer..." }]
    }
  }
}
```

Then stream audio chunks:
* **Audio Input format**: PCM 16-bit, 16kHz, mono (little-endian), wrapped in a base64 string:
  ```json
  {
    "realtimeInput": {
      "mediaChunks": [
        {
          "mimeType": "audio/pcm",
          "data": "BASE64_PCM_CHUNK"
        }
      ]
    }
  }
  ```
* **Audio Output format**: Base64 PCM data returned in the `serverContent` message.

## 7. Migration Roadmap

1. **Phase 1: SDK & Core Logic Upgrade (1-2 Days)**
   * Replace dependencies.
   * Migrate `resume.ts`, `job-description.ts`, `gap-analysis.ts`, and `report-generator.ts` to Gemini REST APIs.
2. **Phase 2: WebSocket Server Setup (2-3 Days)**
   * Set up Fastify proxy server or build client-side direct token WebSocket authentication.
   * Write tools (Function calling) for state persistence and credit tracking.
3. **Phase 3: Frontend Voice Client Integration (3 Days)**
   * Replace browser SpeechRecognition in `voice-chat.tsx` with raw PCM audio recording (using MediaRecorder/Web Audio API).
   * Stream audio input to WebSocket; decode and play PCM audio output.
4. **Phase 4: Testing & Calibration (2 Days)**
   * Calibrate VAD parameters.
   * Validate interruption flows and test state recovery on drop.
